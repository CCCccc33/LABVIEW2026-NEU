# 基于电脑声卡的语音识别与音频信号分析系统说明文档

## 1. 系统概述

本系统基于 LabVIEW 平台设计，利用电脑声卡和麦克风实现音频信号采集、语音命令识别、实时波形显示、频谱分析、音频参数计算以及扩展功能控制。系统可通过语音输入简单命令，例如“分析”“音乐”等，实现对应功能的自动调用。

系统主要由语音识别模块、音频信号分析模块、音频滤波模块、测试音生成模块和扩展控制模块组成。其中，语音识别模块用于接收用户语音命令；音频分析模块用于实时显示音频时域波形和频域频谱，并计算幅值、主频、信噪比等参数；滤波模块用于对音频信号进行低通、高通或带通滤波；测试音模块用于通过声卡输出指定频率的正弦波信号。

## 2. 系统功能

系统实现的主要功能如下：

- 通过 Windows/.NET 语音识别组件识别指定语音命令。
- 通过电脑声卡采集麦克风输入音频信号。
- 实时显示原始音频信号的时域波形。
- 对音频信号进行 FFT 频谱分析并显示频谱图。
- 计算音频信号的幅值、均方根值、主频和信噪比。
- 对音频信号进行低通、高通、带通等滤波处理。
- 显示滤波后音频信号及其频谱。
- 生成指定频率和幅值的正弦波测试音。
- 根据语音命令调用音频分析子 VI 或音乐播放功能。

## 3. 系统总体结构

系统采用主 VI 加子 VI 的结构。主 VI 负责系统界面、登录状态、语音识别初始化和语音命令分发；音频分析相关功能封装在独立子 VI 中，便于维护和扩展。

系统结构如下：

```text
主 VI
  -> 用户登录/注销
  -> 语音识别初始化
  -> 语音命令接收
  -> 命令分支处理
       -> “分析”：调用音频信号分析仪子 VI
       -> “音乐”：调用音乐播放器或打开外部播放器
       -> “测试”：调用测试音生成模块

音频信号分析仪子 VI
  -> 声卡输入采集
  -> 时域波形显示
  -> FFT 频谱分析
  -> 幅值、主频、SNR 计算
  -> 音频滤波
  -> 测试音输出
```

## 4. 语音识别模块设计

语音识别模块调用 Windows 提供的 .NET 程序集 `System.Speech.dll`，主要使用以下类：

```text
System.Speech.Recognition.SpeechRecognitionEngine
System.Speech.Recognition.Choices
System.Speech.Recognition.GrammarBuilder
System.Speech.Recognition.Grammar
```

语音识别流程如下：

```text
创建 SpeechRecognitionEngine
-> 创建 Choices 关键词列表
-> 创建 GrammarBuilder
-> 创建 Grammar
-> LoadGrammar 加载语法
-> SetInputToDefaultAudioDevice 设置默认麦克风
-> Register Event Callback 注册识别事件
-> RecognizeAsync(Multiple) 开始连续识别
```

系统设置的语音命令可包括：

```text
你好
早上好
中午好
下午好
分析
测试
音乐
```

当识别到语音后，通过回调 VI 获取识别结果 `e.Result.Text`，再使用 Case Structure 对不同命令进行分支处理。例如：

```text
分析 -> 打开音频分析子 VI
音乐 -> 打开音乐播放器或网易云音乐
测试 -> 启动正弦波测试音模块
```

## 5. 音频采集模块设计

音频采集模块使用 LabVIEW 声音输入相关 VI，通过电脑声卡采集麦克风输入信号。主要流程如下：

```text
Sound Input Configure
-> Sound Input Start
-> While Loop
     -> Sound Input Read
     -> Get Waveform Components
     -> 输出 Y 数组和 dt
-> Sound Input Stop
-> Sound Input Clear
```

其中：

- `Y` 表示当前采集帧的音频幅值数组。
- `dt` 表示采样时间间隔。
- 采样频率 `fs = 1 / dt`。

建议采样参数如下：

```text
采样率 fs = 44100 Hz
每帧采样点数 N = 2048 或 4096
声道数 = 1
```

系统采用分帧实时分析方式，即每采集一帧音频数据，就立即进行波形显示、FFT 分析和参数计算。

## 6. 频谱分析模块设计

频谱分析用于将音频信号从时域转换到频域，观察信号中不同频率成分的分布。系统采用 FFT 实现频谱分析。

处理流程如下：

```text
音频 Y 数组
-> 加窗处理
-> FFT 频谱分析
-> 输出 magnitude 和 df
-> 显示频谱图
```

其中：

```text
df = fs / N
```

`df` 表示频谱图中相邻频点之间的频率间隔，`N` 为 FFT 点数。

主频计算方法为：

```text
主频 = 最大幅值所在索引 k * df
```

为避免直流分量影响，可在查找最大频谱峰值时跳过 0 Hz 附近的前几个频点。

## 7. 音频参数计算

系统主要计算以下音频参数。

### 7.1 幅值

幅值可使用峰值幅值、峰峰值或 RMS 表示。

```text
峰值幅值 = max(abs(Y))
峰峰值 = max(Y) - min(Y)
RMS = sqrt(mean(Y^2))
```

LabVIEW 中可使用 `Abs`、`Array Max & Min`、`RMS.vi` 等节点实现。

### 7.2 主频

主频由 FFT 频谱计算得到：

```text
magnitude -> Array Max & Min -> max index
主频 = max index * df
```

该结果表示当前音频帧中能量最强的频率成分。

### 7.3 信噪比 SNR

信噪比用于描述信号与噪声的强弱关系。系统可先在安静环境下采集一段噪声基准，得到噪声 RMS：

```text
Noise RMS = RMS(Y_noise)
```

实时采集时计算当前信号 RMS：

```text
Signal RMS = RMS(Y_current)
```

信噪比计算公式为：

```text
SNR(dB) = 20 * log10(Signal RMS / Noise RMS)
```

为避免除以 0，可在噪声 RMS 后加一个很小的数，例如：

```text
Noise RMS + 1E-9
```

## 8. 音频滤波模块设计

系统使用 LabVIEW 的 Butterworth Filter 实现音频滤波，可根据需要选择低通、高通、带通或带阻滤波。

常见滤波类型如下：

```text
Lowpass：低通滤波，保留低频，抑制高频噪声
Highpass：高通滤波，保留高频，抑制低频干扰
Bandpass：带通滤波，只保留指定频率范围
Bandstop：带阻滤波，抑制指定频率范围
```

针对语音信号，推荐默认使用带通滤波：

```text
滤波类型：Bandpass
低截止频率 fl = 100 Hz
高截止频率 fh = 4000 Hz
滤波阶数 = 4
采样率 fs = 44100 Hz
```

滤波模块结构如下：

```text
Filter Button = True
  原始信号 -> Butterworth Filter -> 滤波后信号 -> 滤波后频谱

Filter Button = False
  原始信号直接用于分析
  滤波后图表可清零或保持空显示
```

为了便于观察滤波效果，前面板可同时显示原始波形、原始频谱、滤波后波形和滤波后频谱。

## 9. 正弦波测试音模块设计

测试音模块用于生成指定频率的正弦波，并通过声卡输出。该功能可用于测试声卡输出、验证频谱分析是否正确。

主要流程如下：

```text
设置测试音频率和幅值
-> 生成正弦波数组
-> Sound Output Configure
-> Sound Output Write
-> Sound Output Clear
```

前面板建议设置：

```text
输出设备 ID
测试音频率
测试音幅值
播放/停止按钮
```

需要注意的是，声卡输入设备 ID 和输出设备 ID 是分开编号的。例如输入设备 ID 可能为 0，而输出设备 ID 可能为 1。若出现错误 4800，通常表示选择的输出设备无效，需要切换输出设备 ID。

## 10. 音调识别扩展功能

音调识别用于根据音频主频判断其接近的音符。例如：

```text
440 Hz -> A4 / La
261.63 Hz -> C4 / Do
```

实现流程如下：

```text
音频采集
-> FFT 或主频检测
-> 得到主频
-> 与标准音符频率表比较
-> 输出最接近的音名
```

该功能更适合识别正弦波、乐器音或哼唱音，对于普通说话声只能得到近似主频。

## 11. 音乐播放器扩展功能

音乐播放器可作为语音控制扩展功能。当语音识别结果为“音乐”时，系统可执行以下操作之一：

```text
打开外部音乐软件
或
打开 LabVIEW 内置音乐播放器子 VI
```

简单实现方式是调用外部播放器，例如打开网易云音乐。若使用 LabVIEW 内置播放器，建议优先支持 WAV 文件：

```text
选择 WAV 文件
-> Read WAV File
-> Sound Output Configure
-> Sound Output Write
-> Stop/Clear
```

## 12. 子 VI 设计建议

音频分析子 VI 建议设计为独立运行的界面型子 VI，主 VI 只负责打开和运行它。子 VI 可命名为：

```text
Audio Analyzer.vi
或
声卡音频信号分析仪.vi
```

子 VI 前面板建议包含：

```text
输入设备 ID
输出设备 ID
采样率
采样点数
开始/停止按钮
原始时域波形图
原始频谱图
滤波后时域波形图
滤波后频谱图
峰值幅值
RMS
主频
SNR
滤波开关
滤波类型
低截止频率
高截止频率
测试音频率
测试音幅值
```

子 VI 内部建议采用 While Loop 加 Case Structure 的结构，实现实时采集和状态切换。

## 13. 使用方法

1. 打开主 VI。
2. 设置输入设备 ID 和输出设备 ID。
3. 初始化语音识别模块。
4. 点击开始识别或运行主程序。
5. 对麦克风说出语音命令。
6. 说“分析”时，系统打开音频分析子 VI。
7. 在音频分析界面点击 Start，开始采集和分析音频。
8. 根据需要开启滤波，调节滤波类型和截止频率。
9. 查看时域波形、频谱图、幅值、主频和 SNR。
10. 若使用测试音功能，设置输出设备 ID、测试音频率和幅值后播放。

## 14. 常见问题

### 14.1 Sound Output Configure.vi 报错 4800

该错误表示选择的输出设备无效。解决方法：

```text
检查 Windows 默认输出设备是否可用
尝试修改输出设备 ID，例如 -1、0、1、2
确认输入设备 ID 和输出设备 ID 没有混用
重启 LabVIEW 后重新运行
```

### 14.2 语音识别没有反应

可能原因：

```text
未安装中文语音识别语言包
麦克风输入设备不可用
未调用 RecognizeAsync
未注册 SpeechRecognized 回调事件
关键词没有加入 Choices
```

### 14.3 滤波前后差别不明显

可能原因是原始语音本身主要集中在 100 Hz 到 4000 Hz 范围内。可通过调整截止频率观察更明显效果，例如：

```text
Lowpass cutoff = 800 Hz
Highpass cutoff = 2000 Hz
Bandpass = 100 Hz ~ 1000 Hz
```

## 15. 总结

本系统利用 LabVIEW 的图形化编程能力和 Windows .NET 语音识别组件，实现了一个基于电脑声卡的简单语音识别与音频信号分析系统。系统能够通过语音命令调用音频分析功能，并对麦克风采集到的音频信号进行实时波形显示、FFT 频谱分析、幅值计算、主频检测、信噪比计算和滤波处理。同时，系统还扩展了正弦波测试音生成、音调识别和音乐播放等功能，具有较好的综合性和可扩展性。
